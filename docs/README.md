**第一次了解这个项目**：按 00 → 01 → 02 → 03 → 04 顺序读。

**只想看"做什么"**：读 [00](./00-overview.md) + [01](./01-content-plan.md)。

**只想看"怎么做"**：读 [02](./02-tech-stack.md) + [04](./04-roadmap.md)。

## 文档维护原则

1. **文档先行**：任何新功能/改动，先在这里落字，再写代码。
2. **决策留痕**：技术选型的"为什么"要写清楚，方便日后回顾。
3. **保持更新**：方案变了就改文档，别让文档和代码脱节。
