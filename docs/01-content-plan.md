# 内容规划 · honlnk-home

> 首页各板块的详细方案。调性：**克制、有记忆点、让作品说话**。

## 整体结构

单页滚动，从上到下 5 个板块：

```
┌─────────────────────────────────────────┐
│  1. Hero 首屏                            │  ← 第一印象，3秒定去留
├─────────────────────────────────────────┤
│  2. 关于我 About                         │  ← 简短，不啰嗦
├─────────────────────────────────────────┤
│  3. 精选作品 Projects        ⭐核心⭐     │  ← 占最大篇幅
├─────────────────────────────────────────┤
│  4. 技能栈 Skills                        │  ← 轻量化标签
├─────────────────────────────────────────┤
│  5. 联系方式 Contact + Footer            │
└─────────────────────────────────────────┘
```

## 1. Hero 首屏

### 布局示意

```
               [ GitHub 头像 ]
                  鸿影
                  honlnk

         独立开发者 / AI 应用全栈 · 济南

      "This loser is lazy and leaves nothing behind."

      [ 查看作品 ]   [ GitHub ]   [ 博客 ]
```

### 内容

| 元素 | 内容 | 说明 |
|---|---|---|
| 头像 | GitHub 头像（圆形裁切）| 源：`avatars.githubusercontent.com/u/235359418` |
| 主名 | **鸿影** | 最大字号 |
| 副标 | honlnk | 小字，英文 ID，等宽字体 |
| 定位 | 独立开发者 / AI 应用全栈 · 济南 | 一句话，不喊口号 |
| 座右铭 | "This loser is lazy and leaves nothing behind." | 保留个人 bio，强记忆点 |
| CTA | 查看作品 / GitHub / 博客 | 3 个按钮，不超过 3 个 |

### 设计要点

- 头像在顶部，圆形，呼应 GitHub 身份。
- 名字最大，视觉中心。
- **保留那句自嘲 bio 当座右铭**——这是辨识度，比俗套自我吹捧强 10 倍。
- CTA 按钮最多 3 个，多了分散注意力。
- **背景纯色静态，不加动态背景**（不喜欢花里胡哨）。

## 2. 关于我 About

### 内容（控制在 3-4 行正文 + 标签）

> **一段话**：
> 我是鸿影，一个在济南的独立开发者。热衷于把 AI 能力做成**能真正用起来的产品**——从图像生成、长篇写作到自托管 MCP 工具链。前端用 TypeScript/Vue，后端靠 Java，Python 当胶水。

> **标签**：
> `#AI应用` `#全栈` `#独立开发` `#MCP生态` `#自托管`

### 设计要点

- 不要写成小作文。
- 标签只放 5 个最核心的，不堆砌。
- 强调"**做成产品**"而非"我会什么技术"——有 22⭐ 项目的人，底色是产品型选手。

## 3. 精选作品 Projects ⭐

这是整个首页的灵魂。**不要列全部 44 个仓库**，只精选 5-6 个旗舰。

### 展示卡片（初版精选清单）

| 序号 | 项目 | 描述 | 技术栈 | Star | 链接 |
|---|---|---|---|---|---|
| 1 | **GPT Image Studio** | 本地优先的 AI 图片创作工作台 | TypeScript | 22⭐ / 3⑂ | image.honlnk.com |
| 2 | **NovAI** | 长篇小说 AI 写作工作空间（RAG 驱动）| TypeScript | 5⭐ / 1⑂ | novai.honlnk.com |
| 3 | **linkseek** | 自托管 MCP 服务（联网搜索 + 网页抓取）| TypeScript | 1⭐ | mcp.honlnk.com |
| 4 | **text-diff-viewer** | 纯文本差异对比工具 | TypeScript | 3⭐ | diff.honlnk.top |
| 5 | **GradeSieve** | 成绩筛选工具 | Vue | - | grade.honlnk.com |
| 6 | **dundun（顿顿）** | AI 烹饪私教 | Vue | - | cook.honlnk.com(待绑) |

### 卡片字段规范

每张卡片统一包含：

| 字段 | 说明 |
|---|---|
| 项目名 | 大字标题 |
| 一句话描述 | 精炼版 description |
| 技术栈标签 | 1-2 个主语言 |
| Star / Fork 数 | 社交证明（可动态拉取或手填）|
| 访问链接 | 指向已绑子域名 |
| 悬停动效 | 卡片轻微上浮 + 阴影 |

### 排序与展示逻辑

- **有独立域名的项目优先**（image/novai/grade），点开就能体验，转化率最高。
- 卡片网格布局：桌面 2-3 列，移动端单列。
- 底部一个"在 GitHub 查看全部 →"链接。

## 4. 技能栈 Skills

轻量化展示，**不画进度条**（"Java 90%"那种很土）。用分类标签：

```
前端    TypeScript · Vue3 · Vite · Tailwind
后端    Java · Spring Boot · Python
AI/工具 MCP · RAG · LLM 应用
运维    Docker · Nginx · Cloudflare · Linux
```

### 设计要点

- 四行足矣，或做成四列图标墙。
- 重点是"我会什么能做出东西"，不是炫技清单。

## 可选加分板块（看精力，首版不做）

| 板块 | 价值 | 建议 |
|---|---|---|
| 🕐 最近更新 | 拉最近 push 的仓库，展示活跃度 | 二期再说 |

> 🌙 **深色/亮色主题切换**已从"可选"升级为**首版必做**（见 [设计规范](./03-design-system.md#主题模式-已定亮暗双主题切换)）。

## 内容维护原则

1. **作品卡片是动态的**——每完成一个新项目，加一张卡片（这本身就是复盘仪式）。
2. **描述要精炼**——每张卡片描述不超过 15 字。
3. **Star 数定期更新**——手填或构建时拉 API，保持真实。
